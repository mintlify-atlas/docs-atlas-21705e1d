## Overview

The `useMyLocation` hook provides a simple interface to access the user's current geographic location using the browser's Geolocation API. It returns a Leaflet `LatLng` object containing the user's coordinates, with built-in error handling and high-accuracy positioning.

## Implementation

```typescript
import { useState, useEffect } from "react"
import { LatLng } from "leaflet"

export const useMyLocation = () => {

    const [position, setPosition] = useState<LatLng | null>(null);

    useEffect(() => {

        navigator.geolocation.getCurrentPosition(
            (pos) => {
                const { latitude, longitude } = pos.coords;
                setPosition(new LatLng(latitude, longitude));
            }, () => {
                alert("No se pudo obtener tu ubicación");
            }, {
            enableHighAccuracy: true,
            timeout: 5000,
            maximumAge: 0
        }
        );

    }, [])


    return {
        position
    }
}
```

## Parameters

This hook does not accept any parameters.

## Return Value

The hook returns an object with the following property:

<ResponseField name="position" type="LatLng | null">
  The user's current geographic position as a Leaflet `LatLng` object, or `null` if the position hasn't been retrieved yet.
  
  The `LatLng` object contains:
  - `lat`: Latitude coordinate
  - `lng`: Longitude coordinate
</ResponseField>

### Return Type

```typescript
{
  position: LatLng | null
}
```

## Geolocation API Configuration

The hook uses the following Geolocation API options:

<ParamField path="enableHighAccuracy" type="boolean" default="true">
  Requests the most accurate position available, which may take longer and consume more power.
</ParamField>

<ParamField path="timeout" type="number" default="5000">
  Maximum time (in milliseconds) the device is allowed to take to return a position. Set to 5 seconds.
</ParamField>

<ParamField path="maximumAge" type="number" default="0">
  Maximum age of a cached position (in milliseconds). Set to 0 to always request a fresh position.
</ParamField>

## Permissions and Error Handling

<Warning>
  The Geolocation API requires user permission to access location data. The browser will prompt the user for permission when this hook is first used.
</Warning>

If the geolocation request fails (due to permission denial, timeout, or other errors), the hook displays an alert with the message: "No se pudo obtener tu ubicación" (Could not get your location).

The `position` state will remain `null` if:
- The user denies location permissions
- The geolocation request times out (after 5 seconds)
- The device cannot determine the location
- Geolocation API is not available in the browser

## Usage Example

```tsx
import { useMyLocation } from '@/hooks/useMyLocation'
import { MapContainer, TileLayer, Marker } from 'react-leaflet'

function LocationComponent() {
  const { position } = useMyLocation()

  if (!position) {
    return <div>Loading your location...</div>
  }

  return (
    <MapContainer center={position} zoom={13}>
      <TileLayer
        url="https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png"
      />
      <Marker position={position} />
    </MapContainer>
  )
}
```

### Advanced Usage with Conditional Rendering

```tsx
import { useMyLocation } from '@/hooks/useMyLocation'

function MyLocationButton() {
  const { position } = useMyLocation()

  const handleCenterMap = () => {
    if (position) {
      // Center map on user's location
      map.flyTo(position, 15)
    }
  }

  return (
    <button 
      onClick={handleCenterMap}
      disabled={!position}
    >
      {position ? 'Center on My Location' : 'Getting location...'}
    </button>
  )
}
```

## Browser Compatibility

<Note>
  The Geolocation API is supported in all modern browsers, including:
  - Chrome 5+
  - Firefox 3.5+
  - Safari 5+
  - Edge (all versions)
  - Opera 10.6+
</Note>

### HTTPS Requirement

<Warning>
  Most modern browsers require a secure context (HTTPS) to use the Geolocation API. The API may not work on HTTP pages, except for `localhost` during development.
</Warning>

## Best Practices

1. **Handle null state**: Always check if `position` is `null` before using it
2. **User privacy**: Inform users why you need their location before the browser prompts them
3. **Fallback handling**: Provide alternative functionality if location access is denied
4. **Loading states**: Show appropriate UI feedback while the position is being retrieved
5. **HTTPS**: Ensure your application is served over HTTPS in production
