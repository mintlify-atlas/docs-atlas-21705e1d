# Quick Start Guide

This guide will walk you through starting Mi Ruta Directa and calculating your first route. In less than 5 minutes, you'll have a fully functional routing application running on your machine.

<Info>
  Before starting, make sure you've completed the [installation guide](/installation) and have all dependencies installed.
</Info>

## Start Your First Route

<Steps>
  <Step title="Start the Development Server">
    Navigate to your project directory and start the Vite development server:
    
    <CodeGroup>
      ```bash npm
      npm run dev
      ```
      
      ```bash yarn
      yarn dev
      ```
      
      ```bash pnpm
      pnpm dev
      ```
    </CodeGroup>
    
    You should see output similar to:
    
    ```bash
    VITE v7.2.4  ready in 325 ms
    
    ➜  Local:   http://localhost:5173/
    ➜  Network: use --host to expose
    ➜  press h + enter to show help
    ```
    
    <Note>
      The development server starts on **port 5173** by default. If this port is in use, Vite will automatically try the next available port.
    </Note>
    
    The development server includes:
    - **Hot Module Replacement (HMR)** - Changes appear instantly without full page reload
    - **Fast Refresh** - React components update while preserving state
    - **TypeScript checking** - Real-time type error detection
  </Step>
  
  <Step title="Open the Application">
    Open your web browser and navigate to:
    
    ```
    http://localhost:5173
    ```
    
    You should see the Mi Ruta Directa interface with:
    - An interactive map centered on default coordinates (Barranquilla, Colombia)
    - A search form at the top with three input fields: Origin, Destination, and City
    - A "Buscar" (Search) button
    
    <Tip>
      When you first load the page, your browser will ask for **location permission**. Allow this to see your current location marked on the map.
    </Tip>
    
    ### What You Should See
    
    **Search Form**: Located at the top of the screen with three inputs:
    - **Origin** (MapPin icon) - Starting address
    - **Destination** (Navigation icon) - Ending address  
    - **City** (Building icon) - City name for better geocoding accuracy
    
    **Map View**: An interactive OpenStreetMap showing:
    - Default center location or your current location
    - Zoom and pan controls
    - Full-screen capability
  </Step>
  
  <Step title="Calculate Your First Route">
    Let's calculate a route between two locations. Enter the following example addresses:
    
    <CodeGroup>
      ```text Example 1: Barranquilla, Colombia
      Origin: Calle 72 #50-50
      Destination: Calle 85 #50-20
      City: Barranquilla
      ```
      
      ```text Example 2: Bogotá, Colombia  
      Origin: Carrera 7 #32-16
      Destination: Calle 26 #13-19
      City: Bogotá
      ```
      
      ```text Example 3: Any City
      Origin: [Your starting address]
      Destination: [Your ending address]
      City: [Your city name]
      ```
    </CodeGroup>
    
    Click the **"Buscar"** (Search) button to calculate the route.
    
    ### What Happens Behind the Scenes
    
    When you click "Buscar", the application:
    
    1. **Validates inputs** - Ensures all three fields are filled
    2. **Geocodes addresses** - Converts text addresses to coordinates using Photon API:
       ```typescript
       const origenSeguro = encodeURIComponent(direccionOrigen);
       const resOrigen = await fetch(
         `https://photon.komoot.io/api/?q=${origenSeguro},${ciudad}&limit=1`
       );
       ```
    3. **Calculates route** - Sends coordinates to OSRM for route calculation:
       ```typescript
       const resRuta = await fetch(
         `https://router.project-osrm.org/route/v1/driving/${lonO},${latO};${lonD},${latD}?overview=full&geometries=geojson`
       );
       ```
    4. **Renders on map** - Displays the route as a blue polyline with markers
  </Step>
  
  <Step title="View Your Route">
    After clicking "Buscar", you should see:
    
    **Route Visualization**:
    - A **blue polyline** showing the calculated route between locations
    - **Markers** at the origin and destination points
    - The map automatically **zooms and pans** to fit the entire route in view
    
    **Auto-Fit Behavior**:
    The map uses Leaflet's `fitBounds` to ensure the entire route is visible:
    ```typescript
    map.fitBounds(coordenadas, { padding: [50, 50] });
    ```
    
    <Note>
      The route is calculated using OpenStreetMap data and optimized for driving. The blue line represents the recommended path.
    </Note>
    
    ### Interacting with the Map
    
    You can now:
    - **Pan** - Click and drag to move the map
    - **Zoom** - Use mouse wheel or pinch gestures
    - **Click markers** - View popup information (if available)
    - **View your location** - See a marker showing where you are
  </Step>
  
  <Step title="Calculate Another Route">
    The form inputs are cleared after each search. To calculate another route:
    
    1. Enter new origin, destination, and city
    2. Click "Buscar" again
    3. The map updates with the new route
    
    <Tip>
      Each new route calculation replaces the previous one. The map automatically adjusts to show the new route in full view.
    </Tip>
  </Step>
</Steps>

## Common Issues and Solutions

<Warning>
  **"No se encontró la ubicación"** - This alert appears when the Photon API cannot find the addresses you entered.
</Warning>

<AccordionGroup>
  <Accordion title='"No se encontró la ubicación" Error'>
    **Causes**:
    - Addresses are too vague or incomplete
    - City name doesn't match the address
    - Photon API has no data for that location
    
    **Solutions**:
    - Be more specific with addresses (include street numbers)
    - Verify the city name is correct
    - Try major landmarks or well-known addresses
    - Use full street names (e.g., "Carrera" not "Cra")
  </Accordion>
  
  <Accordion title="Location Permission Denied">
    **Symptoms**: Alert saying "No se pudo obtener tu ubicación"
    
    **Solution**: Grant location permission in your browser:
    - Click the location icon in the address bar
    - Select "Allow" for location access
    - Refresh the page
  </Accordion>
  
  <Accordion title="Map Doesn't Load">
    **Possible causes**:
    - No internet connection (OpenStreetMap tiles require connectivity)
    - Browser blocks external resources
    - Leaflet CSS not loaded
    
    **Solutions**:
    - Check your internet connection
    - Disable browser extensions that block resources
    - Open browser console (F12) to check for errors
  </Accordion>
  
  <Accordion title="Route Shows But Seems Inaccurate">
    **Known limitations** (as noted in README):
    - Addresses sometimes appear near but not exact
    - Destination may be placed in a different city
    - Photon geocoding has varying accuracy by region
    
    **Workarounds**:
    - Use more specific addresses
    - Include nearby landmarks in your search
    - Try alternative address formats
  </Accordion>
</AccordionGroup>
