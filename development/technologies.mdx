## External APIs

The application integrates with two public APIs:

<CardGroup cols={2}>
  <Card title="Photon Geocoding API" icon="map-location-dot">
    **Provider:** Komoot
    
    **Base URL:** `https://photon.komoot.io/api/`
    
    **Purpose:** Convert street addresses to coordinates
    
    **Features:**
    - Free, no API key required
    - OpenStreetMap data
    - Supports multiple languages
    
    **Usage:**
    ```typescript
    const res = await fetch(
      `https://photon.komoot.io/api/?q=${address},${city}&limit=1`
    );
    ```
  </Card>
  
  <Card title="OSRM Routing API" icon="route">
    **Provider:** Project OSRM
    
    **Base URL:** `https://router.project-osrm.org/`
    
    **Purpose:** Calculate driving routes between coordinates
    
    **Features:**
    - Free, open-source routing
    - Based on OpenStreetMap data
    - Returns GeoJSON geometry
    
    **Usage:**
    ```typescript
    const res = await fetch(
      `https://router.project-osrm.org/route/v1/driving/
      ${lon1},${lat1};${lon2},${lat2}?
      overview=full&geometries=geojson`
    );
    ```
  </Card>
</CardGroup>

<Warning>
  Both APIs are public services without authentication. They may have:
  - Rate limiting
  - Availability constraints
  - Data accuracy limitations
  
  See [Known Issues](/development/known-issues) for details.
</Warning>

## Browser APIs

### Geolocation API

Used in `useMyLocation` hook:

```typescript
navigator.geolocation.getCurrentPosition(
  (pos) => {
    const { latitude, longitude } = pos.coords;
    setPosition(new LatLng(latitude, longitude));
  },
  () => alert("No se pudo obtener tu ubicación"),
  {
    enableHighAccuracy: true,  // GPS if available
    timeout: 5000,             // 5-second timeout
    maximumAge: 0              // Don't use cached position
  }
);
```

**Browser Support:**
- Chrome/Edge: ✅ Full support
- Firefox: ✅ Full support
- Safari: ✅ Full support (requires HTTPS)

<Info>
  Geolocation requires HTTPS in production and user permission to access location.
</Info>

## Technology Choices

### Why These Technologies?

<AccordionGroup>
  <Accordion title="React 19 - Latest Features">
    - Concurrent rendering for better UX
    - Automatic batching reduces re-renders
    - Improved hydration
    - Future-proof with latest React patterns
  </Accordion>
  
  <Accordion title="TypeScript - Type Safety">
    - Catch errors at compile time
    - Better IDE autocomplete
    - Self-documenting code with interfaces
    - Easier refactoring
  </Accordion>
  
  <Accordion title="Vite - Fast Development">
    - Instant server start (vs. webpack)
    - Lightning-fast HMR
    - SWC compiler for 20x faster builds

  <Accordion title="Tailwind v4 - Rapid UI Development">
    - Utility-first approach
    - No context switching between files

    - Smaller bundle with JIT compiler
